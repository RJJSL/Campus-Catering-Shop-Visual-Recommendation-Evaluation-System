## 1 概述

### 1.1 用户简介

#### 1.1.1 游客

| 核心需求 | 获取美食推荐信息、查看店铺基本信息和评价 |

#### 1.1.2 用户

| 属性 | 说明 |
|------|------|
| 角色定义 | 已注册并登录的学生用户 |

| 属性 | 说明 |

### 1.2 项目目的与目标

#### 1.2.1 项目目的

"今天吃什么？"这是每个学生每天都会面临的灵魂拷问。校园周边美食店铺众多，但缺乏一个集中、直观、便捷的平台帮助学生快速做出选择。本小程序旨在为学生提供一个便捷的美食发现工具：当学生不知道吃什么的时候，打开小程序，能够获得美食灵感，发现附近的美味，轻松做出选择。

#### 1.2.2 项目目标

| 目标 | 说明 |
|------|------|
| 美食发现 | 帮助学生快速找到想吃的美食，解决决策困难 |
| 真实评价 | 建立学生间的真实评价体系，提供可信参考 |
| UGC共建 | 鼓励学生上传店铺和菜品，丰富平台内容 |
| 便捷体验 | 微信内使用，无需下载安装，扫码即用 |

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| UGC | 用户生成内容（User Generated Content），指学生上传的店铺、菜品、评价等 |
| 随机推荐 | 小程序核心功能，随机匹配一家店铺推荐给用户 |
| 评分分布 | 展示店铺或菜品的星级评分比例分布 |
| 菜品互动 | 用户对菜品进行点赞、收藏、评论的行为 |
| 新店推荐 | 展示近期学生上传的新店铺 |

## 6 系统其他需求

### 6.1 安全性

| 需求 | 说明 |
|------|------|
| 登录安全 | 微信授权登录，防止恶意登录 |
| 数据加密 | 用户敏感信息加密存储 |
| 接口验证 | 所有接口进行身份验证 |
| 防篡改 | 对关键数据进行签名验证 |
| 内容审核 | 对用户上传内容进行审核机制 |
| 防止攻击 | 防止SQL注入、XSS等常见攻击 |

| 需求 | 说明 |
|------|------|
