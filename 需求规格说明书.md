## 2 项目概述

### 2.1 产品背景

"今天吃什么？"这是每个学生每天都会面临的灵魂拷问。校园周边美食店铺众多，但缺乏一个集中、直观、便捷的平台帮助学生快速做出选择。本项目旨在开发一款校园美食推荐小程序，为学生提供便捷的美食发现工具。

### 2.2 用户角色定义

【AI辅助】本部分由ChatGPT根据用户调研生成角色定义初稿，经本人修改完善。

#### 2.2.1 游客

| 属性 | 说明 |
|------|------|
| 角色定义 | 未注册/未登录的用户 |
| 用户标识 | 微信匿名访问用户 |
| 权限范围 | 浏览店铺列表、查看店铺详情、查看评价内容 |
| 核心需求 | 获取美食推荐信息、查看店铺基本信息和评价 |

#### 2.2.2 注册用户

| 属性 | 说明 |
|------|------|
| 角色定义 | 已注册并登录的学生用户 |
| 用户标识 | 通过微信授权登录的用户 |
| 权限范围 | 完整使用所有功能，包括评价、收藏、上传等 |
| 核心需求 | 获取美食灵感、分享用餐体验、上传店铺和菜品 |

#### 2.2.3 管理员

| 属性 | 说明 |
|------|------|
| 角色定义 | 系统管理人员 |
| 用户标识 | 后台管理系统登录用户 |
| 权限范围 | 管理店铺、管理评价、管理用户、数据统计 |
| 核心需求 | 维护平台内容质量、处理违规内容、统计分析 |

### 2.3 目标与范围

#### 2.3.1 项目目标

| 目标 | 说明 |
|------|------|
| 美食发现 | 帮助学生快速找到想吃的美食，解决决策困难 |
| 真实评价 | 建立学生间的真实评价体系，提供可信参考 |
| UGC共建 | 鼓励学生上传店铺和菜品，丰富平台内容 |
| 便捷体验 | 微信内使用，无需下载安装，扫码即用 |

#### 2.3.2 项目范围

| 类型 | 内容 |
|------|------|
| 功能范围 | 店铺推荐、店铺浏览、店铺详情、搜索筛选、用户评价、收藏管理、店铺上传 |
| 用户范围 | 湖南工业大学在校学生 |
| 地域范围 | 校园周边美食店铺 |
| 平台范围 | 微信小程序 |
