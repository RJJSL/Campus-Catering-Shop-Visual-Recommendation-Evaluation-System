## 1 引言

### 1.1 编写目的

在完成了校园美食店铺可视化推荐与评价小程序需求分析和概要设计的基础上，依据《需求规格说明书》和《概要设计说明书》对项目进行了详细设计。本说明书旨在将项目的详细设计方案文档化，为后续编码实现提供详细指导，确保开发团队能够按照统一的设计规范进行开发工作。

### 1.2 术语定义

| 术语 | 定义 |
|------|------|
| UGC | 用户生成内容（User Generated Content），指学生上传的店铺、菜品、评价等 |
| 随机推荐 | 小程序核心功能，随机匹配一家店铺推荐给用户，帮助选择困难症用户做决定 |
| 评分分布 | 展示店铺或菜品的星级评分比例分布，以可视化图表形式呈现 |
| 菜品互动 | 用户对菜品进行点赞、收藏、评论的行为 |
| 新店推荐 | 展示近期学生上传的新店铺，帮助用户发现新美食 |
| 游客 | 访问小程序但还未注册或未进行登录的用户，仅可浏览部分内容 |
| 用户 | 已通过微信授权登录的在校学生，可使用完整功能 |
| 管理员 | 负责维护平台秩序的特殊用户，可管理店铺、评价等 |

### 5.1 登录/注册页面

**包含功能模块**：登录注册及个人信息维护子系统。

**页面参数**：

| 参数名 | 参数说明 |
|--------|----------|
| code | 微信授权临时凭证，用于换取openid |
| openid | 用户在微信生态系统中的唯一标识 |
| session_key | 微信会话密钥，用于解密用户敏感数据 |

**调用背景**：当游客访问小程序时，如果需要使用收藏、评价、上传等用户专属功能，系统会自动跳转到登录页面。或者游客也可以主动点击首页的“微信登录”按钮进入该页面。

- 页面背景：展示美食主题的渐变背景图，营造轻松愉快的氛围

- 用户协议链接：在登录按钮下方提供用户协议和隐私政策的链接
- 登录状态提示：显示当前登录状态和登录进度

**页面使用简介**：

用户首次打开小程序时，如果本地没有有效的登录态，页面会自动显示登录引导界面。用户点击“微信登录”按钮后，小程序会调用微信授权接口，获取用户的授权。授权成功后，系统会使用授权码向后端服务器发起登录请求。后端服务器使用授权码向微信服务器换取用户的openid，然后查询数据库中是否存在该用户，如果不存在则自动创建新用户。登录成功后，系统会保存Token到本地存储，并跳转到首页。

### 5.2 首页推荐页面

**包含功能模块**：店铺浏览子系统、登录注册及个人信息维护子系统。

**页面参数**：

| 参数名 | 参数说明 |
|--------|----------|
| shop_id | 店铺唯一标识，用于加载店铺详情 |
| recommend_type | 推荐类型标识，用于区分随机推荐、今日推荐、新店推荐 |
| latitude | 用户纬度坐标，用于计算店铺距离 |
| longitude | 用户经度坐标，用于计算店铺距离 |

**调用背景**：当用户成功登录后，首先进入的就是首页推荐页面。即使是未登录的游客，也可以浏览首页的部分推荐内容，但无法使用收藏、评价等互动功能。

**页面组成**：

- 顶部导航栏：包含小程序Logo、搜索入口、用户头像入口
- 随机推荐区：醒目的“随便吃啥”按钮，采用随机动画效果推荐店铺
- 今日推荐轮播：自动轮播展示今日热门推荐的店铺卡片
- 新店推荐列表：横向滚动的最新上传店铺列表
- 分类快捷入口：餐厅、小吃、饮品、甜点等分类图标
- 附近美食列表：按距离排序的附近店铺列表，支持下拉刷新和上拉加载更多

### 5.3 店铺浏览页面

**包含功能模块**：店铺浏览子系统。

**页面参数**：

| 参数名 | 参数说明 |
|--------|----------|
| area_id | 区域筛选条件，如南苑、西苑、北苑、东苑 |
| type_id | 类型筛选条件，如餐厅、小吃、饮品、甜点 |
| sort_by | 排序方式，可选值：rating（评分）、heat（热度）、distance（距离） |
| keyword | 搜索关键词 |
| page | 分页页码 |
| page_size | 每页数量，默认20条 |

**调用背景**：用户可以通过点击首页的分类入口、搜索结果，或者在任意页面点击“浏览店铺”入口来进入店铺浏览页面。该页面支持灵活的筛选和排序功能，帮助用户找到符合条件的店铺。

- 筛选标签栏：横向滚动的筛选标签，包括区域筛选、类型筛选
- 排序选项卡：支持按评分、热度、距离排序
- 店铺列表区：展示符合条件的店铺卡片，支持下拉刷新和上拉加载
- 空状态提示：当没有符合条件的店铺时显示的友好提示

### 5.5 评价发表页面

**包含功能模块**：用户交流子系统。

**页面参数**：

| 参数名 | 参数说明 |
|--------|----------|
| shop_id | 店铺唯一标识，关联评价到指定店铺 |
| rating | 星级评分，范围1-5 |

管理员后台是独立于小程序的网页端管理平台，管理员通过专门的登录入口访问。在登录页，管理员需要输入正确的管理员账号和密码进行身份验证。验证通过后会进入管理主页，首页展示了关键的业务统计数据，帮助管理员快速了解平台运行状况。左侧的导航栏提供了各个管理模块的入口。店铺管理模块允许管理员查看所有店铺的列表，可以根据审核状态筛选，查看店铺详情，对违规店铺进行删除操作。评价管理模块允许管理员查看所有用户评价，识别和删除违规评价内容。用户管理模块允许管理员查看用户列表，可以对违规用户进行封禁处理。分类管理和区域管理模块允许管理员维护系统的基础数据。所有的管理操作都会被记录日志，便于后续追溯和审计。
