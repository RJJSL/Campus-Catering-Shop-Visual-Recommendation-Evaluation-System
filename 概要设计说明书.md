## 3 接口设计

### 3.1 接口概述

【AI辅助】本部分由ChatGPT根据RESTful API设计原则生成接口设计初稿，经本人修改完善。

#### 3.1.1 接口规范

| 项目 | 规范说明 |
|------|----------|
| 通信协议 | HTTPS |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |
| 认证方式 | JWT Token |
| 状态码 | 200成功/401未授权/404未找到/500服务器错误 |

#### 3.1.2 通用响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    // 业务数据
  }
}
```

### 3.2 用户接口

| 接口路径 | 方法 | 描述 | 请求参数 | 响应数据 |
|----------|------|------|----------|----------|
| /api/users/login | POST | 用户登录 | openid, nickname, avatar | user, token |
| /api/users/info | GET | 获取用户信息 | - | user |
| /api/users/update | PUT | 更新用户信息 | nickname, avatar, phone | - |

【AI辅助】本表格由本人根据用户模块需求独立设计完成。

### 4.2 数据库逻辑设计

#### 4.2.1 用户表（users）

【AI辅助】本部分由ChatGPT根据数据库设计规范生成表结构初稿，经本人修改完善。

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 用户ID |
| openid | VARCHAR(100) | UNIQUE, NOT NULL | 微信OpenID |
| nickname | VARCHAR(50) | - | 用户昵称 |

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 店铺ID |
| name | VARCHAR(100) | NOT NULL | 店铺名称 |
| image | VARCHAR(255) | - | 店铺图片 |
| rating | DECIMAL(2,1) | DEFAULT 0 | 评分 |
